= XML2JDTO Documentation

This directory contains the complete documentation for the XML2JDTO project in AsciiDoc format.

== Documentation Structure

* `index.adoc` - Main documentation homepage
* `getting-started.adoc` - Installation and basic usage guide
* `api-reference.adoc` - Complete API reference
* `advanced-features.adoc` - Advanced features like schema validation and catalogs
* `examples.adoc` - Real-world usage examples
* `configuration.adoc` - Configuration options and environment setup

== Building the Documentation

=== Prerequisites

* Ruby with AsciiDoctor gem, or
* Docker with AsciiDoctor image

=== Option 1: Using AsciiDoctor (Recommended)

Install AsciiDoctor:

[source,bash]
----
# On Ubuntu/Debian
sudo apt-get install asciidoctor

# On macOS
brew install asciidoctor

# Or using Ruby gems
gem install asciidoctor
----

Build HTML files:

[source,bash]
----
# Build all documentation files
cd docs
asciidoctor *.adoc

# Or build individual files
asciidoctor index.adoc
asciidoctor getting-started.adoc
asciidoctor api-reference.adoc
asciidoctor advanced-features.adoc
asciidoctor examples.adoc
asciidoctor configuration.adoc
----

=== Option 2: Using Maven (if Asciidoctor Maven plugin is configured)

[source,bash]
----
cd ..
mvn generate-resources
----

=== Option 3: Using Docker

[source,bash]
----
docker run --rm -v $(pwd):/documents asciidoctor/docker-asciidoctor \
  asciidoctor *.adoc
----

== Publishing to GitHub Pages

=== Setup GitHub Pages

1. Go to your repository settings on GitHub
2. Navigate to "Pages" section
3. Select source: Deploy from a branch
4. Select branch: `main` (or your default branch)
5. Select folder: `/docs`
6. Click Save

=== Build and Commit

[source,bash]
----
# Build HTML files
cd docs
asciidoctor *.adoc

# Commit generated HTML
git add *.html
git commit -m "Build documentation"
git push
----

Your documentation will be available at:
`https://debugoss.github.io/xml2jdto/`

=== Automated Building with GitHub Actions

Consider adding a GitHub Action to automatically build documentation:

**.github/workflows/docs.yml**:
[source,yaml]
----
name: Build Documentation

on:
  push:
    branches: [ main ]
    paths:
      - 'docs/**/*.adoc'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install AsciiDoctor
        run: sudo apt-get install -y asciidoctor
      
      - name: Build documentation
        run: |
          cd docs
          asciidoctor *.adoc
      
      - name: Commit and push
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add docs/*.html
          git commit -m "Auto-build documentation" || exit 0
          git push
----

== Viewing Locally

After building, open any HTML file in your browser:

[source,bash]
----
# On Linux
xdg-open docs/index.html

# On macOS
open docs/index.html

# On Windows
start docs/index.html

# Or use Python's HTTP server
cd docs
python3 -m http.server 8000
# Then open http://localhost:8000
----

== Styling

The documentation uses the default AsciiDoc styling. To customize:

1. Create a custom CSS file: `custom.css`
2. Build with custom stylesheet:

[source,bash]
----
asciidoctor -a stylesheet=custom.css index.adoc
----

== Contributing to Documentation

1. Edit the `.adoc` files (not the `.html` files)
2. Build and preview locally
3. Commit both `.adoc` and generated `.html` files
4. Submit a pull request

== File Format

All documentation is written in AsciiDoc format (.adoc). 

AsciiDoc advantages:
* Clean, readable source
* Rich formatting capabilities
* Easy to version control
* Can generate multiple output formats (HTML, PDF, etc.)
